指标说明

帮助区分「本站服务」「代理出口」「上游官方」三层状态，避免把不同来源的指标混为一谈。

出口IP

每张 OpenAI / Gemini 卡片下方展示该分组当前绑定的代理出口，名称来自 proxies.name，与 管理后台代理库 及 /apps/proxies 中看到的名称一致（完整显示，如 kookeey-US-1）。

展示项 含义
出口IP · 代理名 该分组账号当前绑定的 SOCKS 出口
括号内毫秒 优先显示 TTFB（首字节时间），与 ops 测速表一致

数据链路：sub2api 每 5 分钟写入 ops_proxy_probe_results → VPS cron 每分钟同步为 proxy-health.json → status-view 按分组绑定切片 → 页面每 60 秒轮询展示。与 /admin/ops 代理节点测速 同源同表，最多约 1 分钟延迟；ops 展示全部活跃代理，status 卡片只展示该分组绑定的那一个。

出口IP 四级状态（与 ops 对齐）

状态 判定条件 颜色
正常 探测成功，且 TTFB < 1200 ms 绿色
偏慢 探测成功，且 1200 ms ≤ TTFB ≤ 3000 ms 琥珀色
严重 探测成功但 TTFB > 3000 ms，或缺少 TTFB 样本 红色
失败 探测未成功（链路不可达或超时） 红色

绑定出口为偏慢时，对应卡片会从「正常」降为「降级」；为严重 / 失败时，卡片降为「异常」。

卡片「正常 / 降级 / 异常」

综合 Uptime Kuma 心跳、对话延迟、端点 PING、可用率、上游官方状态与出口IP健康，按优先级判定：

  1. 无最新主心跳 → 未知
  2. 主心跳 Down，或对话延迟 > 15 s，或端点 PING > 2 s → 异常
  3. OpenAI / Gemini 官方降级，或主心跳非 Up，或对话延迟 ≥ 5 s，或端点 PING ≥ 500 ms，或可用率 < 99% → 降级
  4. 其余 → 正常

仅 openai-default / gemini-default 会叠加出口IP判定；出口数据超过 15 分钟未更新时不改卡片状态，避免误报。

上游官方状态横幅

仅当 OpenAI 或 Gemini 官方发生事故时，卡片区上方会出现黄色提示条（平时隐藏，避免与卡片状态重复）。

场景 横幅 卡片
上游无事故 不显示 不因官方状态降级
OpenAI / Gemini 官方事故 显示事故说明 对应卡变黄（即使本站 chat 探测仍绿）
仅本地代理 / 出口问题 不显示 看卡片「出口IP」区与卡片状态

数据来源：OpenAI status.openai.com 状态 API；Gemini Google Cloud incidents.json 中未结束的 Gemini / Vertex 事故。
